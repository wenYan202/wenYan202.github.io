Information Nutrition Labeling for better Imformation Retrieval

10-12-2020

This is a large project and our team focus on implementation to add a new label to classify good news and bad news. It’s a good bad news classifier, and it’s Supervised Learning.

First, we unify the standard of news classification We manually selected 300 English news articles from different countries. The news topic includes world, business, technology, health, and entertainment. And there are three different clusters of good news or bad news and neutral news. These data are organized as a CSV file. We selected half of them randomly and read each news and give it a score manually. Take this news and the final scores as the original data.

Then we filtered the news to remove words such as a the, then we need to find news’ core idea before processing the text information.

Here the method we used first is TF-IDF. This method calculates the TF-IDF value for every word, and the larger the value is, generally speaking, the higher the importance of this word in this article will be, to achieve feature extraction.

Then chi-Squared Test is used to test whether there is a relationship between variables. Because words with high word frequency may not necessarily represent importance, this step is carried out to select the features which were just extracted.

Finally, This data information was represented by the Bag of Words model. A dictionary is built based on the document, and then an n-dimensional array is used to represent the document.

After processing the data, it is then data analyzed using machine learning classification methods.

Here we called four classification methods, Naive Bayes, SVM, Logistic Regression, and LSTM. These four methods are all good classification methods for machine learning, but the results are not the same because of the different algorithm and other reasons like the correlation of the previous data.

The quality of the machine learning method is not only described by the accuracy, here are four main criteria to describe the final machine learning results. It’s obvious that naïve Bayes works well.

Accuracy is the most intuitive performance measure and it is simply a ratio of correctly predicted observation to the total observations. but as for some great percent data it may not describe so well.

Precision is the ratio of correctly predicted positive observations to the total predicted positive observations.
Recall is the ratio of correctly predicted positive observations to the all observations in actual class it will be good for the score more than 0.5.
F1 Score is the weighted average of Precision and Recall. Therefore, this score takes both false positives and false negatives into account. Intuitively it is not as easy to understand as accuracy, but F1 is usually more useful than accuracy, especially if you have an uneven class distribution.
TF-IDF: This method calculates the TF-IDF value for every word, and the larger the value is, generally speaking, the higher the importance of this word in this article will be.
chi-Squared Test: used to test whether there is a relationship between variables. If there is a low result of it then this part may not so useful for model.
Bag of Words model: actually it's a representing method of text data. here a dictionary is built based on the document, and then an n-dimensional array is used to represent the document.
Logistic Regression : It uses a logistic function to frame binary output model. use sigmoid function. The output of the logistic regression will be a probability (0≤x≤1), and can be used to predict the binary 0 or 1 as the output.
Naive bayes is a generative model. Naive bayes works well with small datasets.
SVM :support vector machine works relatively well when there is a clear margin of separation between classes. SVM is more effective in high dimensional spaces.but not suitable for large data sets.
LSTM: Long short-term memory. They are a special kind of Neural Network called Recurrent Neural Networks. but it always has the overfitting problems.
